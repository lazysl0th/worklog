# Журнал работ

Внутренний инструмент для автоматизации учета, контроля и ежедневной фиксации выполненных объемов. Проект реализован в виде монорепозитория.

---

## ⚡ Особенности реализации и стандарты качества

- **Cтрогая типизация:** Полный запрет на использование типа `any` и приведений типов через `as` (Type Assertion). Безопасность типов гарантирована на уровне AST-пайплайна ESLint.
- **Архитектурная изоляция:** Четкое разделение ответственности. Backend спроектирован по принципам Clean Architecture, Frontend — по методологии Feature-Sliced Design (FSD). Границы слоев и направления зависимостей контролируются линтером.
- **Инфраструктура Docker:** Полное локальное окружение (App + DB) поднимается одной командой. Версии баз данных и рантайма зафиксированы в Alpine-контейнерах для предсказуемости деплоя.
- **Data Integrity:** Вся валидация входных данных, бизнес-сущностей и переменных окружения изолирована с помощью схем `Zod`.

---

## 🛠 Технологический стек

### Бэкенд (`backend/`)

- **Платформа:** Node.js (v26), TypeScript, Express
- **База данных & ORM:** PostgreSQL (v18) + Prisma ORM
- **Архитектура:** Clean Architecture (Domain -> Application -> Infrastructure)
- **Валидация данных:** Zod

### Фронтенд (`frontend/`)

- **Фреймворк:** React, TypeScript, Vite
- **Стейт-менеджмент:** Redux Toolkit (RTK Query / Слайсы)
- **Стилизация:** Tailwind CSS
- **Архитектура:** Feature-Sliced Design (FSD)

---

## 🏗 Архитектура проекта

### Backend (Clean Architecture)

Зависимости направлены строго внутрь. Бизнес-логика (`Domain` и `Application`) ничего не знает о базе данных, фреймворках и внешних библиотеках:

- **Domain:** Чистые сущности бизнеса, интерфейсы репозиториев, правила валидации.
- **Application:** Сценарии использования (Use Cases), обработчики команд и запросов.
- **Infrastructure:** Реализация репозиториев (Prisma), контроллеры (Express), конфигурации, внешние интеграции.

### Frontend (Feature-Sliced Design)

Код разделен на слои, каждый из которых имеет строго ограниченную зону видимости:
`app` (инициализация) -> `pages` (страницы) -> `widgets` (композиция) -> `features` (действия пользователя) -> `entities` (бизнес-сущности) -> `shared` (переиспользуемый код).

## 🚀 Быстрый старт

### Требования

- Node.js v20+ / npm v10+
- Docker и Docker Compose

### 1. Установка

npm install

### 2. Запуск инфраструктуры (Docker)

# Запуск базы данных и контейнеров

npm run infra:up

# Полная очистка

npm run infra:clean

### 3. Контроль качества кода

# Запуск линтера

npm run lint

# Форматирование

npm run format

# Сквозная проверка

npm run check

---

## Скрипты управления

- infra:up — Поднять Docker-контейнеры.
- infra:down — Остановить контейнеры.
- lint — Поиск ошибок типизации и архитектуры.
- format — Автоматическое форматирование кода.
- check — Полный цикл проверки (lint + format).
